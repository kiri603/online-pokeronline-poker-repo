## 1. 背景

归心已支持普通要不起和固守式要不起触发重置选择。用户进一步明确：如果桌面牌属于归心玩家，其他玩家因要不起触发【观星】，归心玩家也应可选择发动归心保护并解除禁用。

当前观星流程是：玩家在要不起时先进入 `GUANXING` 响应窗口，完成选牌后才执行等价 PASS 的回合收尾。因此归心判断应放在观星完成之后、回合推进之前。

## 2. 更新目标

- 桌面牌属于禁用归心玩家时，对方因要不起触发【观星】，完成观星后进入归心选择窗口。
- 归心玩家接受后解除归心禁用，对方本次观星式要不起不再追加普通 PASS 摸牌。
- 保持观星原有摸 5 选 2、弃置其余牌和防重复提交逻辑不变。

## 3. 任务类型

问题修复 / 技能联动规则补齐。

## 4. 影响范围

- `poker/src/main/java/com/game/poker/service/GameService.java`
  - 在 `resolveGuanxing(...)` 的 `PASS` 上下文中接入归心待响应判断。
- `poker/src/test/java/com/game/poker/service/GuixinSkillServiceTest.java`
  - 新增观星完成后触发归心待响应并可解除禁用的测试。
- `AI_CODE_GUIDE.md`
  - 补充技能式要不起错误模式中的观星示例。

## 5. 功能需求

1. 玩家因普通要不起触发【观星】时，先正常完成观星选牌。
2. 观星完成后，如果桌面牌拥有者是禁用状态的归心玩家，且观星玩家不是该归心玩家本人，则创建 `GUIXIN_DECISION`。
3. 创建 `GUIXIN_DECISION` 后不得立即 `handleNextTurnAfterPass(...)`。
4. 归心玩家接受保护后，归心解除禁用，回合按要不起语义继续流转。
5. 归心玩家不发动时，观星玩家不再因普通 PASS 额外摸 2 张，因为观星本身已经替代本次要不起惩罚。

## 6. 非功能需求

- 不改变观星主动收益和选牌数量。
- 不改变普通 PASS、固守、借刀杀人和铁骑规则。
- 不新增前端字段；复用现有 `GUIXIN_DECISION` 状态和前端响应 UI。
- 不修改数据库、依赖、认证授权或生产配置。

## 7. 不在本次范围

- 不调整观星技能描述和 UI 样式。
- 不改变归心主动使用规则。
- 不清理无关截图、数据库文件和临时目录。

## 8. 验收标准

1. 对方触发观星并完成选牌后，若桌面牌属于禁用归心玩家，后端进入 `GUIXIN_DECISION`。
2. 归心玩家接受后，`guixinDisabled=false`。
3. 观星玩家手牌只增加观星选中的 2 张，不额外摸普通 PASS 惩罚牌。
4. 归心相关测试通过，相邻技能测试不回归。

## 9. 风险与注意事项

- 归心判断不能放在观星开始前，否则会阻断观星发动。
- 归心判断不能放在回合推进后，否则 `lastPlayPlayerId` 可能被清空，导致无法判断桌面归属。
- WebSocket 在观星完成后已有 `broadcastGameState`，应复用现有状态同步。
