# Codex 项目级工作规则

本文件是 Codex 在 `online-poker-repo` 项目中的入口配置。

## 强制规则

- 以后在本项目中进行任何需求开发、问题修复、重构、配置调整或文档更新时，必须先读取并遵循根目录 `WORKFLOW.md`。
- `WORKFLOW.md` 是本项目更新流程的唯一主规则来源；如果本文件与 `WORKFLOW.md` 有重复描述，以 `WORKFLOW.md` 中更具体、更严格的规则为准。
- 代码生成、问题修复和重构时，必须在 `WORKFLOW.md` 之后读取并遵循根目录 `AI_CODE_GUIDE.md`；该文档负责沉淀项目级代码生成标准、历史错误模式和 Bug 修复复盘规则。
- 不需要用户每次额外提醒，所有修改都默认自动套用 `WORKFLOW.md` 中的执行检查清单、需求文档、确认门禁、实现步骤、测试步骤和完成报告标准。
- 需求文档确认前，只允许做只读分析和需求文档整理，不得修改业务源代码。
- 任何会影响数据库 schema、认证授权、安全策略、生产部署、依赖版本或大规模目录结构的操作，都必须先单独向用户确认。
- 必须保护用户已有未提交改动，不得覆盖、回滚或清理与当前任务无关的文件。

## 每次任务开始时必须做

1. 读取 `WORKFLOW.md`。
2. 读取 `AI_CODE_GUIDE.md`，说明本次遵循的关键代码标准和高风险规则。
3. 按 `WORKFLOW.md` 展示并维护“执行检查清单”。
4. 运行 `git status --short`，说明当前已有改动和本次预计触碰范围。
5. 按任务类型生成或更新 `需求文档/` 下的版本化需求文档。
6. 等用户确认需求文档后，再进入代码或配置修改。
7. 如本次是 Bug 修复，完成后必须判断是否要把错误模式沉淀进 `AI_CODE_GUIDE.md`，并在完成报告中说明。

## 项目固定背景

- 前端：`Vue 3 + Vite`，目录为 `poker-frontend/`。
- 后端：`Java 25 + Spring Boot + Maven`，目录为 `poker/`。
- 需求文档目录：`需求文档/`。
- 项目更新流程文档：`WORKFLOW.md`。
- AI 代码生成与修复标准文档：`AI_CODE_GUIDE.md`。

## 默认回复语言

除非用户明确要求英文，否则所有说明、计划、报告和总结都使用简体中文。
